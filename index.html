Dasar Teori

CSS (Cascading Style Sheets), menggambarkan bagaimana elemen-elemen HTML ditampilkan pada layar, kertas, atau media lainnya. CSS mempersingkat pekerjaan dengan cara mengendalikan tampilan (layout) beberapa halaman web sekaligus. CSS dapat ditambahkan ke dalam elemen HTML dengan 3 (tiga) cara, yaitu:

Inline CSS

Inline CSS diterapkan langsung pada elemen HTML menggunakan atribut style. Cara ini cocok untuk mengatur gaya pada satu elemen, namun tidak efisien jika digunakan pada banyak elemen sekaligus karena sulit dikelola.

Internal CSS

Internal CSS ditempatkan dalam elemen style di bagian head dari halaman HTML. Metode ini bagus untuk mengatur gaya satu halaman, namun tidak efisien untuk situs web dengan banyak halaman.

External CSS

External CSS menggunakan file CSS terpisah yang dihubungkan ke halaman HTML. Metode ini adalah yang paling efisien untuk mengelola gaya di banyak halaman web karena mudah diubah dan dikelola secara terpusat.

Css Fonts

CSS Fonts adalah bagian penting dari CSS yang digunakan untuk mengontrol bagaimana teks ditampilkan di halaman web. Dengan CSS Fonts, Anda dapat menentukan jenis huruf (font-family), ukuran huruf (font-size), ketebalan huruf (font-weight), dan gaya huruf (font-style) untuk elemen teks. Properti seperti font-family memungkinkan Anda memilih dari berbagai jenis font, baik yang sudah tersedia secara default di browser atau font kustom dari layanan seperti Google Fonts. font-size mengatur ukuran teks, sedangkan font-weight menentukan ketebalan huruf dari normal hingga bold atau nilai numerik spesifik. font-style mengontrol apakah teks ditampilkan dalam bentuk normal, italic, atau oblique. Selain itu, CSS juga menyediakan properti line-height untuk mengatur jarak antar baris, yang membantu dalam meningkatkan keterbacaan teks. Dengan kemampuan ini, CSS Fonts memungkinkan desain yang konsisten dan menarik pada berbagai perangkat, memastikan bahwa teks di halaman web Anda tidak hanya mudah dibaca tetapi juga estetis.

Elemen Layout HTML

Element layout HTML digunakan untuk menampilkan isi atau konten dalam berbagai bagian atau kolom pada halaman web, mirip dengan format majalah atau surat kabar. HTML5 memperkenalkan beberapa elemen semantik baru yang membantu mendefinisikan struktur halaman dengan lebih jelas dan teratur. Elemen-elemen ini termasuk:

  • <header>: Digunakan untuk mendefinisikan kepala dokumen atau bagian, sering berisi elemen navigasi dan informasi pengantar.
  • <nav>: Menampung link navigasi yang memungkinkan pengguna untuk berpindah antar bagian di situs.
  • <section>: Menandai bagian atau ruangan dokumen yang terpisah, seringkali dengan heading dan konten terkait.
  • <article>: Berisi konten mandiri yang dapat berdiri sendiri, seperti artikel berita atau posting blog.
  • <aside>: Menyediakan konten tambahan yang relevan dengan konten utama, seperti sidebar atau catatan sampingan.
  • <footer>: Menandai bagian akhir dari dokumen atau section, biasanya berisi informasi hak cipta atau detail kontak.
  • <details>: Menyediakan elemen yang dapat di-expand atau di-collapse, sering digunakan untuk menampilkan informasi tambahan.
  • <summary>: Menjadi heading atau ringkasan dari elemen <details>, menampilkan informasi tentang apa yang tersembunyi di bawahnya.

HTML Forms

HTML Forms adalah elemen penting yang memungkinkan pengguna untuk memasukkan data yang kemudian dapat dikirim ke server untuk diproses. Dengan elemen formulir Anda dapat membuat berbagai jenis input untuk mengumpulkan informasi pengguna. Formulir HTML mendukung berbagai atribut untuk mengontrol pengiriman data, validasi, dan pengaturan antarmuka pengguna. Formulir dapat digunakan untuk berbagai tujuan, seperti pendaftaran pengguna, pengisian survei, atau pemesanan produk.

HTML Keygen

Elemen HTML keygen digunakan untuk menghasilkan sepasang kunci publik dan privat yang dapat digunakan untuk otentikasi dan enkripsi. Biasanya digunakan dalam formulir untuk tujuan keamanan, seperti autentikasi pengguna dalam aplikasi web. Elemen ini telah menjadi deprecated dalam HTML5 karena kekurangan dukungan dan masalah keamanan, dan direkomendasikan untuk menggunakan metode alternatif seperti Web Crypto API.
